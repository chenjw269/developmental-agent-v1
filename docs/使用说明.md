# 使用说明

## 一、管理后台 UI 在哪里看

管理后台是一个**前端页面**，和申请页、测试调用页在同一套前端里。

### 方式一：用浏览器打开前端后点导航

1. 先启动整个系统（见下方「重启命令」）。
2. 在浏览器打开 **http://localhost:3000**（Docker 时前端端口为 3000）。
3. 在页面顶部导航栏点击 **「管理后台」**。

### 方式二：直接打开管理后台地址

在浏览器地址栏输入：

- **http://localhost:3000/admin**

即可直接进入管理后台。

进入后会先看到 **「管理员登录」** 界面，输入你在 `.env` 里配置的 `ADMIN_API_KEY`（管理员密码），点击「登录」。登录成功后会看到：

- **申请列表**：所有 API Key 申请记录，可进行「通过」「拒绝」。
- **已发放 API Key**：已通过的凭证列表，可修改配额、Token 余额、禁用/启用等。

## 二、修改代码后如何重启

按你当前是用 **Docker 跑整套** 还是 **本机分别跑前后端** 选择一种即可。

### 1. 使用 Docker Compose（整套一起跑）

在**项目根目录**执行：

```bash
# 停止并删除当前容器（数据卷保留）
docker compose down

# 重新构建并启动（代码有改动时建议加 --build）
docker compose up -d --build
```

- 只改了一个服务时，可只重建该服务，例如只改了前端：
  ```bash
  docker compose up -d --build frontend
  ```
- 只改了 admin-backend：
  ```bash
  docker compose up -d --build admin-backend
  ```

### 2. 本地开发（前后端分开跑）

每次只改哪个服务，就重启哪个：

| 修改的部分 | 在对应目录执行 | 说明 |
|------------|----------------|------|
| **前端** (frontend) | `cd frontend` → `npm run dev` | 改完保存后 Vite 会自动热更新，一般不用手动重启；若要完全重启就停掉再运行一次 `npm run dev`。 |
| **管理后端** (admin-backend) | `cd admin-backend` → `uvicorn app.main:app --reload --host 0.0.0.0 --port 8000` | 使用 `--reload` 时改代码会自动重载；若要完全重启就停掉再运行一次该命令。 |
| **mock-agent-service** | `cd mock-agent-service` → `uvicorn app.main:app --reload --host 0.0.0.0 --port 8000` | 同上，改代码后自动重载或手动重启。 |

前端默认访问：**http://localhost:3000**，管理后台：**http://localhost:3000/admin**。

## 三、小结

| 问题 | 答案 |
|------|------|
| 管理后台在什么界面看？ | 浏览器打开 **http://localhost:3000/admin**，或打开 http://localhost:3000 后点导航「管理后台」。 |
| 修改后怎么重启？ | **Docker**：项目根目录执行 `docker compose down` 再 `docker compose up -d --build`（或只重建改动的服务）。**本地开发**：只重启你改动的服务（前端 `npm run dev`，后端 `uvicorn ... --reload`）。 |
